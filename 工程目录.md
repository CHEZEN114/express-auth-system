# Express 用户认证系统 - 工程目录结构

## 📋 项目概述

**项目名称**: kimi-use  
**技术栈**: Express.js + LowDB/MongoDB + EJS  
**功能**: 完整的用户认证解决方案，支持注册、登录、密码重置、2FA、OAuth、管理后台等

## 🔧 核心文件说明

### 1. 入口文件

**app.js** - 应用程序主入口
- Express 应用初始化
- 中间件配置（Session、Flash、Passport、i18n）
- 全局模板变量设置
- 路由挂载
- 错误处理
- 服务器启动

### 2. 数据库层

**db.js** - LowDB 本地 JSON 数据库
- 纯 JSON 文件存储，无需外部数据库
- 用户 CRUD 操作
- 密码重置令牌管理
- 邮箱验证令牌管理
- 登录日志记录
- API Token 管理

**db-mongodb.js** - MongoDB 数据库兼容层
- Mongoose 模型封装
- 与 db.js 相同的接口，可无缝切换
- 支持生产环境部署

### 3. 配置文件

| 文件 | 说明 |
|------|------|
| config/database.js | 数据库连接配置 |
| config/passport.js | GitHub/Google OAuth 策略配置 |
| config/email.js | SMTP/SendGrid 邮件服务配置 |
| config/i18n.js | 多语言国际化配置 |
| config/redis.js | Redis 缓存连接配置 |
| config/security.js | 安全策略配置（密码强度、登录锁定等） |
| config/swagger.js | API 文档配置 |
| config/upload.js | 文件上传配置 |

### 4. 中间件

**middleware/auth.js**
- `requireAuth` - 需要登录才能访问
- `requireGuest` - 已登录用户不能访问（登录页等）
- `requireRole(role)` - 角色权限检查

**middleware/security.js**
- `checkAccountLock` - 检查账户是否被锁定
- `recordLoginFailure` - 记录登录失败
- `validatePasswordPolicy` - 密码策略验证
- `recordLoginLog` - 记录登录日志

**middleware/admin.js**
- `requireAdmin` - 需要管理员权限
- `checkUserActive` - 检查用户账户状态

### 5. 路由层

| 路由文件 | 功能描述 |
|----------|----------|
| routes/auth.js | 用户注册、登录、登出、密码重置、邮箱验证 |
| routes/oauth.js | GitHub/Google OAuth 登录回调 |
| routes/oauth-wechat.js | 微信 OAuth 登录 |
| routes/user.js | 用户资料查看和修改 |
| routes/security.js | 登录日志查看、安全设置 |
| routes/twofa.js | 双因素认证（2FA）启用/禁用/验证 |
| routes/admin.js | 管理后台（用户管理、系统统计） |
| routes/analytics.js | 数据分析（用户增长、登录统计） |
| routes/api/ | RESTful API 接口 |

### 6. 数据模型 (Mongoose)

**models/User.js**
```javascript
字段说明:
- username: 用户名（唯一，3-30字符）
- email: 邮箱（唯一，小写）
- password: 密码（bcrypt 加密，不返回）
- isEmailVerified: 邮箱是否验证
- role: 角色（user/moderator/admin）
- isActive: 账户状态（启用/禁用）
- avatar: 头像路径
- twoFactorSecret: 2FA 密钥（不返回）
- twoFactorEnabled: 2FA 是否启用
- oauth: OAuth 关联信息
- lastLoginAt: 最后登录时间
```

**models/PasswordResetToken.js** - 密码重置令牌
**models/EmailVerificationToken.js** - 邮箱验证令牌
**models/LoginLog.js** - 登录日志

### 7. 服务层

**services/emailService.js**
- 发送验证邮件
- 发送密码重置邮件
- 支持 SMTP/SendGrid/控制台三种模式
- HTML 邮件模板

### 8. 视图模板

**基础页面**
- `views/index.ejs` - 首页（登录/注册入口）
- `views/dashboard.ejs` - 用户中心仪表盘
- `views/profile.ejs` - 个人资料编辑

**认证页面 (`views/auth/`)**
- `login.ejs` - 登录表单
- `register.ejs` - 注册表单
- `forgot-password.ejs` - 忘记密码表单
- `reset-password.ejs` - 重置密码表单
- `verify-email.ejs` - 邮箱验证结果

**管理后台 (`views/admin/`)**
- `dashboard.ejs` - 管理仪表盘（统计卡片）
- `users.ejs` - 用户管理列表（搜索、分页、筛选）
- `user-detail.ejs` - 用户详情和编辑
- `stats.ejs` - 系统统计图表

**安全页面 (`views/security/`)**
- `2fa-setup.ejs` - 2FA 启用流程
- `2fa-manage.ejs` - 2FA 管理
- `2fa-login.ejs` - 登录时 2FA 验证
- `login-logs.ejs` - 登录历史记录

**公共组件 (`views/partials/`)**
- `head.ejs` - HTML head（CSS、Meta）
- `navbar.ejs` - 响应式导航栏
- `footer.ejs` - 页脚
- `messages.ejs` - Flash 消息提示

## 🚀 运行指南

### 安装依赖
```bash
npm install
```

### 配置环境变量
```bash
cp .env.example .env
# 编辑 .env 文件配置相关参数
```

### 初始化管理员
```bash
node scripts/init-admin.js
# 默认管理员: admin@example.com / Admin@123456
```

### 启动服务
```bash
# 开发模式（自动重启）
npm run dev

# 生产模式
npm start

# 运行测试
npm test
```

### 访问地址
- 首页: http://localhost:3000
- 管理后台: http://localhost:3000/admin
- API 文档: http://localhost:3000/api-docs

## 🔄 数据库切换

### 使用 LowDB（默认，适合开发和测试）
```bash
# 无需额外配置，数据存储在 database.json
npm start
```

### 使用 MongoDB（生产环境）
```bash
# 1. 设置环境变量
export USE_MONGODB=true
export MONGODB_URI=mongodb://localhost:27017/express-auth

# 2. 迁移数据
node scripts/migrate-to-mongodb.js

# 3. 启动
npm start
```

## 🌐 API 接口

### 认证 API
```
POST   /api/v1/auth/register     # 注册
POST   /api/v1/auth/login        # 登录
GET    /api/v1/auth/me           # 获取当前用户
POST   /api/v1/auth/password     # 修改密码
POST   /api/v1/auth/2fa/enable   # 启用 2FA
POST   /api/v1/auth/2fa/disable  # 禁用 2FA
POST   /api/v1/auth/2fa/verify   # 验证 2FA
```

### 管理 API
```
GET    /api/v1/admin/users       # 用户列表
GET    /api/v1/admin/users/:id   # 用户详情
PUT    /api/v1/admin/users/:id   # 更新用户
DELETE /api/v1/admin/users/:id   # 删除用户
GET    /api/v1/admin/stats       # 系统统计
GET    /api/v1/admin/analytics   # 数据分析
```
